# vkr-docx

> **Скилл для Claude Code.** Установка — одна команда:
> ```bash
> curl -fsSL https://raw.githubusercontent.com/twint1m/vkr-docx/main/install.sh | bash
> ```
> **Не клонируй репо вручную** — используй install.sh.

Скилл автоматически оформляет выпускную квалификационную работу (ВКР) в `.docx` по ГОСТ 7.32-2017.

## Что умеет

Ты скидываешь текст главы и картинки — AI оформляет всё по ГОСТу:

- Стили, шрифты, поля, отступы — Times New Roman 14pt, межстрочный 1.5
- Заголовки глав, подразделов, пунктов с правильным форматированием
- Рисунки с автомасштабированием и подписями
- Таблицы с границами и подписями
- Двусторонние кликабельные перекрёстные ссылки (текст ↔ подпись)
- Содержание с leader dots
- Версионирование файлов (`ВКР 0.A.B.docx`)

## Как пользоваться

### 1. Установка (один раз)

Скажи Claude Code:

> установи скилл с https://github.com/twint1m/vkr-docx

Или выполни вручную:

```bash
curl -fsSL https://raw.githubusercontent.com/twint1m/vkr-docx/main/install.sh | bash
```

### 2. Добавление главы

Подготовь:
- **Текст главы** в markdown (`chapter3.md`)
- **Картинки** в папке с именами `fig-1.png`, `fig-2.png`, ...

Скажи Claude Code:

> вот текст главы chapter3.md и картинки в папке images/, добавь в ВКР

AI выполнит:
```python
from vkr_docx import VKRDocument
vkr = VKRDocument("ВКР 0.2.7.docx")
vkr.add_chapter("chapter3.md", images_dir="images/")
vkr.save()  # → ВКР 0.2.8.docx
```

### 3. Исправление стилей

> исправь стили в ВКР

### 4. Обновление ссылок и содержания

> обнови ссылки и содержание в ВКР

## Формат текста главы (markdown)

```markdown
# 2 ПРОЕКТИРОВАНИЕ ВЕБ-ПРИЛОЖЕНИЯ

## 2.1 Алгоритм работы

Текст параграфа. Выравнивание по ширине, отступ 1.25 см.

На рисунке 1 показан алгоритм авторизации.

*Рисунок 1 — Алгоритм авторизации*

Результаты представлены в таблице 1.

*Таблица 1 — Описание полей*

| Поле | Тип | Описание |
|------|-----|----------|
| id   | UUID | Первичный ключ |
| name | VARCHAR | Имя |
```

## Именование картинок

Положи в одну папку:

| Имя файла | Что получится |
|-----------|---------------|
| `fig-1.png` | Рисунок 1 (ширина до 15 см) |
| `fig-14a.png` + `fig-14b.png` | Два рисунка рядом |
| `рис-3.png` или `3.png` | Тоже работает |

Скриншоты мобильных указывай через `screenshot_nums` — они вставятся шириной ~8 см.

## Конфигурация (опционально)

Создай `config.yaml` в рабочей папке, чтобы изменить параметры ГОСТ:

```yaml
page:
  margin_left_cm: 3.0
  margin_right_cm: 1.5
  margin_top_cm: 2.0
  margin_bottom_cm: 2.0

font:
  name: "Times New Roman"
  size: 14

working_dir: "~/Downloads"
```

По умолчанию настроено по ГОСТ 7.32-2017 — менять не обязательно.

## Версионирование

| Версия | Значение |
|--------|----------|
| `ВКР 0.A.B` | В работе. A = глава, B = правки |
| `ВКР 1.0.0` | Готова |
| `ВКР 1.0.N` | Правки после готовности |
